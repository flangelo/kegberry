# kegberry — Claude Code context

## What this is
Deployment repo for Kegbot on a Raspberry Pi. Contains the docker-compose stack, nginx config, kiosk systemd service, and a one-shot install script. The application code lives in separate repos (`kegbot-server`, `kegbot-pycore`).

## Repo layout
| Path | Purpose |
|------|---------|
| `docker-compose.yml` | Defines all services for the kegberry stack |
| `nginx.conf` | Reverse proxy config (HTTP + WebSocket) |
| `db/` | Custom MariaDB image — extends `mariadb:10.11` with timezone table init |
| `kegbot-kiosk.service` | Systemd service that opens Chromium in kiosk mode on the Pi desktop |
| `install.sh` | One-shot bootstrap script for a fresh Pi |
| `bin/kegberry` | Helper script (thin wrapper) |

## Docker services
| Container | Image | Role |
|-----------|-------|------|
| `kegberry-kegbot-1` | `ghcr.io/flangelo/kegbot-server:latest` | Django app (Daphne ASGI) |
| `kegberry-kegnet-listener-1` | `ghcr.io/flangelo/kegbot-server:latest` | kegnet Redis → WebSocket bridge |
| `kegberry-workers-1` | `ghcr.io/flangelo/kegbot-server:latest` | RQ background workers |
| `kegberry-pycore-1` | `ghcr.io/flangelo/kegbot-pycore:latest` | pycore event processor |
| `kegberry-kegboard-1` | `ghcr.io/flangelo/kegbot-pycore:latest` | kegboard serial daemon |
| `kegberry-nginx-1` | `nginx:alpine` | Reverse proxy on port 8000 |
| `kegberry-redis-1` | `redis:7.2-alpine` | Message bus + RQ task queue |
| `kegberry-mysql-1` | built from `db/` | MariaDB database with timezone tables |

## Environment variables (.env)
Generated by `install.sh` on first run. Never committed.

| Variable | Purpose |
|----------|---------|
| `KEGBOT_SECRET_KEY` | Django secret key |
| `KEGBOT_API_KEY` | Shared API key between pycore and kegbot |
| `KEGBOARD_DEVICE` | Path to Arduino device (e.g. `/dev/serial/by-id/usb-...` or `/dev/ttyACM0`) |

## Common operations
```bash
# Deploy updated images (run on Pi)
cd ~/kegberry && docker compose up -d

# Rebuild the db image after changes to db/
docker compose build mysql && docker compose up -d mysql

# Tail logs
docker logs -f kegberry-kegbot-1
docker logs -f kegberry-kegnet-listener-1

# Check all containers
docker ps

# Restart a single service
docker compose restart kegbot
```

## db/ folder — why it exists
Django uses `CONVERT_TZ()` for date queries. If the `mysql.time_zone` tables are empty you get:
```
ValueError: Database returned an invalid datetime value. Are time zone definitions for your database installed?
```
The `db/` folder builds a custom image on top of `mariadb:10.11` that installs `tzdata` and runs `mysql_tzinfo_to_sql` via `/docker-entrypoint-initdb.d/` on first container init.

**If upgrading an existing volume** (init script won't re-run), populate manually:
```bash
docker exec kegberry-mysql-1 bash -c "mysql_tzinfo_to_sql /usr/share/zoneinfo | mysql -u root mysql"
```

## Kiosk service
`kegbot-kiosk.service` runs Chromium in fullscreen kiosk mode pointing at `http://localhost:8000/fullscreen-realtime/`. It is installed as a systemd service under the Pi desktop user.

- `install.sh` installs and enables it automatically
- The `_KIOSK_USER_` placeholder is replaced with the current user at install time
- Chromium uses a throwaway profile in `/tmp/chromium-kiosk` so it always starts clean
- Do **not** run Chromium on the Pi outside this service — it triggers a GNOME keyring popup that hangs the UI

## Fresh Pi install
```bash
curl -sSL https://raw.githubusercontent.com/flangelo/kegberry/master/install.sh | bash
```
The script: checks distro, installs Docker + Chromium, clones this repo, creates the data dir, generates `.env`, pulls images, starts all services, and installs the kiosk systemd service.
